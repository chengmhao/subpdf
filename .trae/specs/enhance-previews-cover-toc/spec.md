# 导出预览增强与封面/目录配置 Spec

## Why
当前“封面页/目录页”只有开关，没有可配置面板；同时实时预览更像网页卡片，不像 Word/PDF 的真实阅读/编辑体验，导致用户难以在导出前确认最终成品。

## What Changes
- 新增“封面页设置”与“目录页设置”两个独立配置面板，仅在对应开关启用时可编辑
- 实时预览升级为“双预览体系”：DOC 预览（Word/Office 风格）与 PDF 预览（PDF 阅读器/打印风格）
- 导出格式选择为 DOC 或 PDF 时，只展示对应预览；选择两者时展示双 Tab 切换
- **BREAKING**：实时预览 DOM 结构将拆分为两个预览容器（原单一预览容器不再作为唯一渲染目标）

## Impact
- Affected specs: 导出模板配置 | 封面/目录结构 | 实时预览 | DOC/PDF 渲染差异展示
- Affected code: index.html | styles.css | script.js

## ADDED Requirements
### Requirement: 封面页独立配置
系统 SHALL 在“文档规格/结构”区域提供“封面页设置”面板，仅当用户启用封面页时该面板可交互。

#### Scenario: 启用封面页并配置
- **WHEN** 用户勾选“封面页”
- **THEN** 显示“封面页设置”面板，面板项可编辑
- **AND** 预览中的封面页内容实时更新

#### Scenario: 未启用封面页
- **WHEN** 用户取消勾选“封面页”
- **THEN** “封面页设置”面板隐藏或禁用
- **AND** 预览中不显示封面页

**封面页设置项（最小集合）**
- 封面标题（默认绑定文件命名）
- 副标题（可空）
- 元信息行（例如：来源文件、日期/版本，可开关）
- 对齐方式（居中/左对齐）
- 风格（极简/品牌色）

### Requirement: 目录页独立配置
系统 SHALL 在“文档规格/结构”区域提供“目录页设置”面板，仅当用户启用目录页时该面板可交互。

#### Scenario: 启用目录页并配置
- **WHEN** 用户勾选“目录页”
- **THEN** 显示“目录页设置”面板，面板项可编辑
- **AND** 预览中目录页实时更新（包含目录标题、层级与引导点）

#### Scenario: 未启用目录页
- **WHEN** 用户取消勾选“目录页”
- **THEN** “目录页设置”面板隐藏或禁用
- **AND** 预览中不显示目录页

**目录页设置项（最小集合）**
- 目录标题（默认“目录”）
- 展示层级（1–3）
- 引导点（显示/隐藏）
- 编号风格（无/1. 2. 3./I II III）

### Requirement: DOC 与 PDF 双预览
系统 SHALL 提供 DOC 预览与 PDF 预览两种视觉呈现，分别体现 Office/Word 与 PDF 阅读器的差异。

#### Scenario: 选择导出 DOCX
- **WHEN** 输出格式为 DOCX
- **THEN** 仅显示 DOC 预览

#### Scenario: 选择导出 PDF
- **WHEN** 输出格式为 PDF
- **THEN** 仅显示 PDF 预览

#### Scenario: 选择导出 DOCX + PDF
- **WHEN** 输出格式为 DOCX + PDF
- **THEN** 同时提供 DOC / PDF 两个预览 Tab，用户可切换

**DOC 预览视觉要求（最小集合）**
- 带 Office/Word 风格的工具条/状态栏
- 页面画布更像“可编辑文档”，表现为：纸张边界、页边距提示、较柔和阴影
- 文本排版采用“文档编辑”风格的排版（例如更自然的字距/行距、可见标题层级）

**PDF 预览视觉要求（最小集合）**
- 带 PDF 阅读器风格工具条
- 页面画布更像“打印/阅读成品”，表现为：更强的纸张阴影、固定布局感
- 文本排版采用“成品阅读”风格（例如更紧凑的渲染、不同的字体/抗锯齿策略模拟）

## MODIFIED Requirements
### Requirement: 实时预览架构
系统 SHALL 将实时预览从单一渲染容器扩展为两个预览容器（DOC/PDF），并以导出格式驱动显示策略。

### Requirement: 导出格式选择行为
系统 SHALL 在用户切换导出格式时同步更新：
- 当前可见预览类型（DOC/PDF/双 Tab）
- 预览顶部标签信息（格式、纸张、方向）

## REMOVED Requirements
无
